# Team Management

OpenInspection supports multiple inspectors and agents under one workspace. Roles control what each user can see and do.

---

## Roles

| Role | Who uses it | Capabilities |
|---|---|---|
| `owner` | Original account creator | Everything |
| `admin` | Senior team members | Everything (same as owner) |
| `inspector` | Field inspectors | Create inspections, collect field data, upload photos |
| `agent` | Real estate agents | View their own referred inspections via agent dashboard |

Roles are assigned at invite time and can be changed directly in D1.

---

## Inviting a Team Member

Team invitations are generated by the portal (InspectorHub SaaS) or can be created directly via the D1 database for standalone deployments.

### Via the Portal (SaaS plan)

Log in to your portal workspace and go to **Team → Invite Member**. Enter the email address and select a role. An invitation link is emailed to the new member.

### Via the API (Standalone)

Use the invite endpoint from the dashboard or directly via curl:

```bash
curl -X POST https://your-domain.com/api/admin/invite \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email": "new.inspector@example.com", "role": "inspector"}'
```

Response includes the invite link and expiry:
```json
{ "success": true, "inviteLink": "https://your-domain.com/join?token=…", "expiresAt": "…" }
```

If `RESEND_API_KEY` is configured, the invite email is sent automatically.

### Via D1 (Standalone — manual fallback)

Insert an invite record directly using `wrangler d1 execute` if the API is unavailable. The `id` column serves as the invite token:

```sql
INSERT INTO tenant_invites (id, tenant_id, email, role, status, expires_at)
VALUES (
    'your-random-uuid-here',
    'your-tenant-id',
    'new.inspector@example.com',
    'inspector',
    'pending',
    unixepoch() + 604800
);
```

Share this URL with the new team member:
```
https://your-domain.com/join?token=your-random-uuid-here
```

## Accepting an Invitation

The invitee visits `/join?token=...`. The join page:
1. Validates the token (checks it exists and hasn't expired).
2. Asks the new member to set a password.
3. Creates their user account under your tenant.
4. Redirects to the dashboard.

## Listing Team Members

To see all current members and pending invites, call:

```bash
curl -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
     https://your-domain.com/api/team/members
```

This endpoint is served by the portal. For standalone deployments, query D1 directly:

```bash
npx wrangler d1 execute openinspection-db \
  --command "SELECT id, email, role, created_at FROM users WHERE tenant_id = 'your-tenant-id';"
```

---

## Changing a User's Role

Edit the `users` table in D1:

```bash
npx wrangler d1 execute openinspection-db \
  --command "UPDATE users SET role = 'admin' WHERE id = 'user-id';"
```

Role changes take effect on the user's next login (their existing JWT retains the old role until it expires or they log in again).

---

## Removing a Team Member

Delete the user from D1:

```bash
npx wrangler d1 execute openinspection-db \
  --command "DELETE FROM users WHERE id = 'user-id';"
```

> Existing inspections created by this user are not deleted — `inspector_id` remains set but the user record is gone. Their completed inspections and reports remain intact.

## Password Reset

### Self-Service (Email)
If `RESEND_API_KEY` and `SENDER_EMAIL` are configured, users can reset their own passwords:
1. Visit the `/login` page.
2. Click **Forgot password**.
3. Submit the email address associated with the account.
4. A reset link will be emailed (or printed to the console in local development).

### Manual reset via SQL (Emergency)
If you cannot receive email, you can update the `password_hash` column directly in D1. The hash is a SHA-256 hex digest of the plain password:

```bash
# On Linux/macOS, compute the hash:
echo -n "newpassword123" | sha256sum
# → e.g., "ef92b778bafe771e89245b89ecbc08a44a4e166c06659911881f383d4473e94f"

npx wrangler d1 execute openinspection-db --remote \
  --command "UPDATE users SET password_hash = 'ef92b778bafe...' WHERE email = 'user@example.com';"
```

## Agent Accounts

Agents (real estate agents who refer clients) get read-only access to their referred inspections via the **Agent Dashboard** at `/agent-dashboard`.

When creating an inspection, set `referredByAgentId` to the agent's user ID to associate the referral. Agents can only see their own inspections; admins and owners can see all referrals and the leaderboard.

See [Booking System — Agent Referral CRM](./06_booking_system.md#agent-referral-crm) for details.
